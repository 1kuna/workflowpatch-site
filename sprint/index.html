Acceptance Checks

Before Payment

  1. Buyer confirms the workflow can be reduced to one first slice.
  2. Buyer confirms the source data and destination boundary.
  3. Buyer accepts the exclusion list and approval rules.

Before Acceptance

  1. Valid examples create the expected output rows or draft records.
  2. Invalid examples create blocked/error rows instead of silent failure.
  3. Customer-facing output remains approval-gated unless the buyer separately accepts a live-send scope.

Guarantee Requests

No guaranteed revenue, rankings, deliverability, compliance outcome, or zero-failure promise. The deliverable is a tested patch with evidence.

Useful first slice: a redacted sample, one current manual step, and one destination where a review queue or ledger would already save time.
