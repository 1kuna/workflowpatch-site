What To Send

Required

  1. Company and website.
  2. The workflow you want patched.
  3. The current source and desired destination.
  4. What happens manually today.
  5. What breaks if it stays manual.
  6. Whether redacted sample data is available within 24 hours.

Helpful

  1. Two example inputs that should succeed.
  2. One example input that should be blocked or reviewed.
  3. Known duplicate, missing-field, or API failure cases.
  4. Who approves any customer-facing output.
  5. The deadline that actually matters.

What Happens Next

1. Fit Check

Alex reviews the workflow async and replies with the smallest version that can be built and tested cleanly.

2. Written Scope

If it fits, you receive a fixed one-source, one-transform, one-destination scope before payment.

3. Build And Handoff

Work starts after payment and access/data readiness. Delivery includes QA evidence, logging, and a runbook.

No implementation starts from the free fit check. Payment links are sent only after the written scope is accepted.
